README - موقع حراجي

وصف
-----
هذا مشروع واجهة أمامية بسيطة لموقع "حراجي" لبيع الأشياء المستعملة. التصميم يهدف لواجهة يمنية جذابة ويحتوي على:

- صفحة رئيسية (`index.html`) مع ترويسة يمنية/عدنية.
- صفحة إضافة منتج (`add.html`) حيث يجب على البائع رفع 3 إلى 20 صورة، وصف، رقم الهاتف، والعنوان.
- صفحة عرض المنتجات (`browse.html`) لعرض منتجات البائعين.

التخزين المحلي
----------------
المنتجات تُخزَّن مؤقتًا في `localStorage` في شكل مصفوفة. هذا مثال للواجهة الأمامية فقط؛ لو تريد تخزينًا دائمًا يجب إضافة API/قاعدة بيانات.

تشغيل (واجهة محلية مع API)
---------------------------------
الخيار السهل (بدون خادم):
1. افتح الملفات `index.html`, `add.html` أو `browse.html` في متصفح حديث (Chrome/Edge/Firefox).
2. املأ الحقول وارفع صوراً (3 إلى 20). هذه الطريقة تخزن البيانات محلياً في `localStorage`.

الخيار الأفضل: تشغيل الخادم المحلي (يوفر رفع صور وواجهة API حقيقية)
1. تأكد أن لديك Node.js مثبتاً.
2. افتح PowerShell في مجلد المشروع (`/haraji`) ثم نفّذ:

```powershell
npm install
npm startرشح
```

3. افتح المتصفح واذهب إلى `http://localhost:3000/add.html` لإضافة منتجات أو `http://localhost:3000/browse.html` لعرضها.

ملاحظات:
- عند تشغيل الخادم تُخزّن الصور داخل مجلد `uploads/` وسجل المنتجات في `products.json`.
- ملف `products.json` و `uploads/` مدرَجان في `.gitignore` لمنع رفعهما إلى المستودع.

ملاحظات مستقبلية
-----------------
- إضافة backend (API + DB) لحفظ المنتجات بصورة دائمة.
- تحسين رفع الصور عبر ضغط أو رفع إلى سحابة.
- إضافة تسجيل دخول للبائعين.

ملفّات المشروع
---------------
- `index.html` - الصفحة الرئيسية
- `add.html` - نموذج إضافة المنتج
- `browse.html` - عرض المنتجات
- `styles.css` - أنماط المشهد
- `script.js` - منطق الواجهة (تحقق، حفظ، عرض)
